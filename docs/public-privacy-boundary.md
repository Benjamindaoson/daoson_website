# 公开隐私边界

## 不公开信息

- 学校
- 专业
- 公司
- 精确工作时间
- 手机
- 微信
- 地址
- 完整简历
- 证书编号
- 客户名称
- 内部项目名称
- 可推断完整职业轨迹的信息

## 允许公开信息

- AI 工程方向
- 系统案例的公开部分
- 工程方法
- 技术判断
- 写作
- GitHub
- Email
- 公开的项目边界

## 可匿名化信息

- 项目中涉及的用户、客户、合作方
- 未公开仓库中的实现细节
- 具体评测样本
- 运行日志中的敏感参数

## 禁止虚构内容

- 学历
- 任职经历
- 公司
- 客户
- 用户数
- 证书
- 奖项
- 业绩结果
- 商业影响

## 当前风险位置

### 页面正文

- `about.md`：会公开技术定位和方法，不能补学校/公司/经历
- `index.html`：会公开研究主线，不能补敏感履历
- `now.md`：会公开当前研究方向，不能补公司或职业履历
- `projects.html` / `_data/projects.yml`：只能写已验证的项目边界

### Front Matter

- `_config.yml`：公开邮箱和 GitHub 是可以的
- `_posts/*.md`：不能出现未经验证的客户、公司、学校、奖项
- `_notes/*.md` / `_til/*.md`：不能把草稿写成履历

### 生成物

- `_site/`：需要检查是否把站点正文之外的信息意外写进去
- `feed-*.xml`：检查摘要是否泄漏敏感信息
- `sitemap.xml`：检查 URL 是否包含不该公开的文件名
- Pagefind 索引：检查搜索是否暴露已降级内容

### 图片与附件

- `assets/img/avatar.jpg`：可作为公开身份图片，但应确认不是私人敏感照片
- `assets/img/covers/*`：应只包含公开内容相关素材
- 本仓库当前未发现 PDF 简历

### Git 历史

已做的历史文件名扫描结果：

- 未发现 `resume`
- 未发现 `pdf`
- 未发现 `school` / `education`
- 未发现 `company`
- 未发现 `linkedin`

说明：这是文件名级扫描，不等于对所有历史内容做了全文审计。

## 结论

这个站点的隐私边界应该是：

**公开技术身份，保留职业联系，拒绝公开履历。**
