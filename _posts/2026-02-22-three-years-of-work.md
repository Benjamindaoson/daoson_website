工作三年了。

如果让我说这三年学到的最重要的一件事，那应该是：
**「保持简单」比想象中难得多**。

## 简单不是少做

刚入行的时候，我以为简单就是「代码少」。所以我会用一堆三元表达式、
箭头函数、链式调用，把十行代码压缩成一行。

后来才明白，简单不是「字符少」，而是「概念少」。

```javascript
// 字符少，但概念多
const result = items.reduce((a, b) => a.set(b.k, (a.get(b.k) || 0) + 1), new Map());

// 字符多，但每一步都清楚
const counts = new Map();
for (const item of items) {
  const current = counts.get(item.k) || 0;
  counts.set(item.k, current + 1);
}
```

后者多了几行，但读的时候不需要"解码"，反而更"简单"。

## 简单的本质是「克制」

写代码三年，最大的成长不是学会了什么新东西，
而是学会了「**不做某些事**」：

- 不做过度抽象
- 不做超前设计
- 不引入用不上的依赖
- 不为了优雅而牺牲清晰
- 不在还没需求时就准备好「扩展点」

每一次克制都需要勇气，因为它意味着——
当未来真的有需求时，可能要返工。

但十次假想的扩展性需求中，真正发生的可能只有一两次。
为这一两次提前付出的"复杂度税"，往往得不偿失。

## 小结

简单 = 克制 + 清晰 + 谦逊。

谦逊指的是：承认你不知道未来会怎么变。
